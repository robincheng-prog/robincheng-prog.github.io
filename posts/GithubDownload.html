在 VS Code 下載 GitHub 倉庫教學

本指南介紹如何配置 Git 環境，並使用 SSH 安全連線來下載（Clone）私有倉庫。

第一步：安裝與基礎配置

在使用 VS Code 與 GitHub 連結之前，系統核心必須先安裝 Git 引擎。

1. 下載並安裝 Git

Windows 使用者請先前往官網下載安裝程式：

👉 Git for Windows 官方下載點

安裝細節說明：

  • Choosing the default editor：建議選擇 “Visual Studio Code” 作為預設編輯器。

  • Adjusting your PATH environment：請務必選擇「Git from the command line and also from 3rd-party software」，這能確保 VS Code 的終端機可以直接讀取到 Git 指令。

  • Line ending conversion：建議選擇預設的 “Checkout Windows-style, commit Unix-style line endings”，以避免跨平台開發時的換行符號錯誤。

如沒有安裝VS Code 可以前往官網下載安裝程式：

👉 VS Code for Windows 官方下載點

安裝完成後，請打開 VS Code 的 Terminal (終端機) 或系統的 CMD。這一步是為了讓 Git 知道是誰在進行提交代碼（Commit）。

第二步：處理私有倉庫驗證

當您存取 私有倉庫 (Private Repo) 時，GitHub 需要驗證您的身分。SSH (Secure Shell) 是一種加密協議，透過成對的「公鑰」與「私鑰」來達成免密登入。

1. 產生 SSH Key 密鑰對

在終端機輸入以下指令來產生金鑰，連續按 Enter 使用預設路徑即可：

  • 參數說明：-t ed25519 是目前最推薦的加密算法（比傳統 RSA 更小更快且更安全）；-C 則是備註，通常寫 Email。

  • 過程提示：指令執行後會詢問三個問題，直接按三次 Enter 使用預設路徑且不設置額外密碼即可。

2. 將公鑰新增至 GitHub 帳戶

您必須將「公鑰」交給 GitHub，而「私鑰」則需鎖在自己的電腦裡。

  1. 取得公鑰內容：

  • 檔案位置：C:\Users\你的用戶名.ssh\id_ed25519.pub (副檔名為 .pub 的才是公鑰)。

  • 您可以直接在終端機輸入 cat ~/.ssh/id_ed25519.pub 來顯示並複製內容。

  1. 上傳至 GitHub：

  • 登入 GitHub，點擊右上角頭像 -> Settings。

  • 左側選單找到 SSH and GPG keys。

  • 點擊 New SSH key，在 “Title” 取名（如：Work-PC），並將剛複製的內容貼入 “Key” 區塊中。

💡 提示：
如私有倉庫是其他人所有，要請代其代為新增公鑰。
如使用多個倉庫就需要多個SSH Key，可以參考 GitHub 獨立金鑰 or GitHub 獨立金鑰 windows 11

第三步：克隆倉庫 (Clone)

現在你可以將遠端專案複製到本地端了。

1. 測試連通性 (解決網路擋牆)

某些公司或學校環境會封鎖 SSH 預設的 22 端口，若連線失敗，請嘗試強制走 443 端口：

若看到 Hi username! You’ve successfully authenticated… 即代表設定完全正確。

2. 執行 Git Clone

回到 GitHub 倉庫頁面，點擊綠色的 Code 按鈕，切換到 SSH 分頁並複製網址（應以 git@github.com… 開頭）。
