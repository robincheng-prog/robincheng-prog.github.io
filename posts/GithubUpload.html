專案上傳到 GitHub

本指南將引導您將本地電腦上的專案程式碼上傳到 GitHub。我們將使用最推薦的 SSH 驗證方式 進行操作。

⚠️ 重要前置檢查

在開始上傳前，請確保您的開發環境已完成以下配置：

  1. 環境安裝：已安裝 Git 客戶端。
  2. 身分識別：已完成 git config 全域帳戶設定。
  3. 安全驗證：這是最關鍵的一步。請確認您已完成 SSH Key 的產生與 GitHub 綁定。

    💡 如果您尚未完成上述步驟，請先閱讀：

    在 VS Code 下載 GitHub 倉庫教學

第一階段：本地專案初始化

請在 VS Code 中開啟您的專案資料夾，並打開終端機 (Terminal)，依照以下步驟操作：

1. 初始化本地 Git 倉庫

將目前的資料夾轉變為 Git 可以追蹤的狀態。

2. 將檔案加入暫存區

您可以選擇加入特定檔案，或使用 . 加入資料夾內所有檔案。

3. 提交變更 (Commit)

這會將檔案正式記錄在本地的 Git 歷史中，-m 後面請輸入本次上傳的說明。

常見問題排錯 (Q&A)

Q1: 出現 “Permission denied (publickey)” 錯誤？

原因：您的電腦沒有正確配置 SSH Key，或者該 Key 沒有新增到 GitHub。

檢查：輸入 ssh -T git@github.com 測試連線。如果失敗，請重新執行 SSH Key 產生與上傳流程。

Q2: 出現 “remote origin already exists” 錯誤？

原因：這代表您之前已經設定過遠端網址。

Q3: 想要忽略某些檔案不要上傳（如：編譯產生的暫存檔）？

解決：在專案根目錄建立一個名為 .gitignore 的檔案，並在裡面列出要忽略的檔案或資料夾名稱（例如：bin/ 或 obj/）。

💡 小撇步 ：

在 VS Code 左側工具列點擊 「原始碼控制」圖示 (Ctrl+Shift+G)，您可以用滑鼠點擊「+」號（等於 git add）和「打勾」圖示（等於 git commit），這比打指令更直觀！
