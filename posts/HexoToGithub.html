hexo上傳到github

💡 提示：

將 Hexo 部落格上傳（部署）到 GitHub Pages，關鍵步驟為:

  • 安裝 hexo-deployer-git 插件。
  • 修改根目錄 _config.yml 的 deploy 設定。
  • 並執行 hexo clean && hexo d 指令。
  • 建議在 GitHub 上建立一個名稱為 <你的使用者名稱>.github.io 的儲存庫以做為個人主頁。

一、 準備工作

建立 GitHub 儲存庫：在 GitHub 建立一個名為 <Username>.github.io 的 Public 儲存庫。
安裝 Git 部署插件：在 Hexo 根目錄下執行以下指令安裝套件：

二、 配置 Hexo

開啟根目錄下的 _config.yml 檔案，在檔案末尾找到並修改 deploy 設定：

注意：

請將<Username>替換為你的 GitHub 使用者名稱。

三、 部署至 GitHub

執行以下指令將產生的靜態檔案發佈到 GitHub：

  • hexo clean：清除舊的靜態檔案（public 資料夾）。
  • hexo g：生成新的靜態檔案。
  • hexo d：部署到 GitHub。

部署到github之前要確認配置 Hexo 的 _config.yml

打開你的 Hexo 博客根目錄下的 _config.yml 文件，修改 deploy 設定：

四、 確認與排錯

查看網站：

部署完成後，即可透過 https://<Username>.github.io 存取網站。

處理自定義網域：若使用自定義網域，在 source 資料夾下建立 CNAME 檔案並填入網域。

錯誤排除：若部署失敗，請確認 SSH Key 是否已設定至 GitHub，或嘗試使用 HTTPS 連結。
