GitHub 獨立金鑰

由於 GitHub 不允許同一個公鑰綁定多個專案，您需要為這兩個專案分別產生不同的 Key（例如 id_rsa_keyA 與 id_rsa_KeyB）。

1. 產生兩組獨立金鑰

  • -t (type): 指定金鑰類型。
  • -f (filename): 指定私鑰存放的檔案路徑和檔名。如果不指定，會使用預設值（通常是 ~/.ssh/id_rsa 或 ~/.ssh/id_ed25519）。
  • -C (comment): 提供一個註解，通常用於辨識使用者（如 email），會顯示在公鑰內容的末尾。
    請將產生的 .pub 公鑰內容分別貼到 GitHub 各自專案的 Settings > Deploy keys 中。

2. 設定 ~/.ssh/config

  • 由於您的 URL 中主機名都是 ://github.com，為了讓 SSH 能根據「路徑」自動選用不同的 Key，最常見的做法是利用 Host 別名。
  • 但如果您堅持要用原生的 URL (不改 alias)，SSH 本身無法根據 URL 的後半段路徑 (<你的github名字/你的專案.git>) 來自動切換金鑰。建議採用以下別名設定方式，並微調您的 Remote URL：
