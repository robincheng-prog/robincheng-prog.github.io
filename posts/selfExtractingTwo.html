自解壓縮執行檔設計方案二

本方案詳述如何將 EDAT 專案中所有的目錄結構與檔案，封裝成一個單一的自解壓安裝檔 (edat_setup.exe)。在執行時，此安裝檔會完整還原工作區的目錄結構，隨後自動調用執行目標批次檔 bin\DC.bat。

由於 IExpress 工具過於老舊，在 modern Windows 10/11 中存在長路徑限制、權限阻擋（需要系統管理員權限編譯與執行）等問題，本方案採用更現代、高相容性且免管理員權限的 Windows 原生 .NET C# 編譯器 (csc.exe) 技術來編譯自解壓安裝檔。

🛠️ 架構設計與運作原理

本方案由三個核心元件協同運作：

  1. build.bat (啟動引導)：在開發環境中執行的入口腳本，負責呼叫 PowerShell 建置核心。
  2. build.ps1 (建置核心)：負責清理舊檔、將工作區打包成 payload.zip、產生 C# 自解壓原始碼並呼叫 csc.exe 編譯成 edat_setup.exe。
  3. payload.zip (壓縮內容)：包含 EDAT 所有程式碼、組件與目錄層級的臨時壓縮檔，會被內嵌至編譯產出的 EXE 資源中。

2. 建置核心腳本 (build.ps1)

使用 PowerShell 編寫，完整建置流程如下：

  • 清理舊檔：清除先前的 payload.zip、Installer.cs 與 edat_setup.exe。
  • 打包工作區：使用 Compress-Archive 將專案內除了 .git、建置腳本與設計文件以外的所有檔案壓縮成 payload.zip。
  • 產生 C# 原始碼：產生 Installer.cs。此程式在執行時會讀取內嵌的 payload.zip 資源，解壓至 %APP_DRIVE%\%APP_DIR%（預設為 C:\scot\dcdat），並執行 bin\DC.bat。
  • 編譯封裝：調用 Windows 內建的 .NET 編譯器 %SystemRoot%\Microsoft.NET\Framework64\v4.0.30319\csc.exe，以 /resource:payload.zip 參數將壓縮檔封裝進執行檔，產出 edat_setup.exe。
  • 暫存清理：自動刪除暫存的 payload.zip 與 Installer.cs。

📂 建議的程式異動

[MODIFY] build.bat

修改為調用 build.ps1 的簡潔引導腳本。

[NEW] build.ps1

新增 PowerShell 建置核心。

🧪 驗證與測試計畫

手動測試驗證

  1. 建置測試：
    • 執行專案根目錄下的 build.bat。
    • 驗證根目錄下是否成功生成 edat_setup.exe，且無任何編譯報錯。
  2. 安裝與還原測試：
    • 將生成的 edat_setup.exe 放置至測試環境中執行。
    • 檢查 C:\scot\dcdat\ 下是否完整還原了 bin/、config/、devices/ 等所有子資料夾，且檔案大小均正確無誤。
  3. 執行測試：
    • 驗證解壓完畢後，是否已自動開啟終端機並正確執行了 bin\DC.bat 及其內部的所有初始化配置。
