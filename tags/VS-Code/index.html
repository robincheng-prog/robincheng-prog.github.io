在 VS Code 中操作 GitHub

在 VS Code 中操作 GitHub，最核心的步驟包含安裝工具、連結帳號、下載專案與提交變更。透過 VS Code 內建的「原始碼控制」面板，即可完成圖形化的版本控制，免去記憶複雜的指令。以下為您整理具體的操作步驟與重點。

1. 事前準備在使用 VS Code 操作 GitHub 之前，您需要先確保電腦中已安裝好以下兩項基本工具：

  • 安裝 Git：前往 Git 官方網站 下載並完成安裝（安裝過程一路點選下一步即可）。
  • GitHub 帳號：若沒有帳號，請至 GitHub 官網 註冊。

2. 將 VS Code 連線至 GitHub

  1. 打開 VS Code，點擊左側活動列的「原始碼控制」圖示（圖示為三個點連接的樹狀結構）。

  1. 點擊「登入」或選擇「從 GitHub 複製」(Clone from GitHub)，這時瀏覽器會自動跳轉至 GitHub 授權頁面。

  2. 登入 GitHub 並點擊授權，授權成功後瀏覽器會詢問是否開啟 VS Code，確認開啟即可完成帳號連結。

3. 下載現有專案 (Clone)

若要將 GitHub 上的專案下載到本機電腦進行修改：

  1. 開啟 VS Code，按下快速鍵 Ctrl + Shift + P（Mac 上為 Cmd + Shift + P）叫出「命令選擇區」。
  2. 輸入 gitcl，並選擇 Git: Clone 指令。
  3. 貼上專案的 GitHub 網址，或直接點選「從 GitHub 複製」來選擇您的專案。
  4. 選擇電腦中的一個資料夾來儲存此專案，下載完成後直接在 VS Code 開啟該資料夾。

4. 提交與同步變更 (Commit & Push)

當您在 VS Code 中修改了專案程式碼後，需將變更上傳至 GitHub：

  1. 查看變更：點擊左側「原始碼控制」圖示，您會看到修改過的檔案。
  2. 暫存變更 (Stage)：將游標移到欲提交的檔案上方，點擊右側出現的「＋」號（或點擊全部變更右側的 + 號）。
  3. 提交 (Commit)：在輸入框中填寫本次修改的「提交訊息」（例如：更新首頁版面），接著點擊上方的「✓」勾勾圖示。
  4. 同步 (Push)：點擊面板下方的「同步變更」按鈕，或點擊右上角的「…」選單選擇「推送」(Push)，即可將程式碼同步上傳到 GitHub。

5. 擴充功能推薦

若要獲得更強大的 GitHub 整合功能，建議在 VS Code 安裝以下擴充套件：

  • GitHub Pull Requests and Issues：直接在編輯器內審查 PR、管理 Issues。
  • GitLens — Git supercharged：提供強大的程式碼歷史紀錄、誰在什麼時候修改了哪一行程式碼等詳細資訊。
