了解 GitHub 的 pull request 工作流程

Hello World

按照這個「Hello World」練習，了解 GitHub 的 pull request 工作流程。

簡介

本課程培訓 GitHub 的基本知識，例如儲存庫、分支、提交和拉取請求等。 你將創建自己的Hello World儲存庫並了解 GitHub的拉取請求工作流程，這是建立和查看程式碼的熱門方法。

在本快速入門指南中，您將：

  • 建立和使用儲存庫。
  • 啟動和管理新分支。
  • 對文件進行更改並將其作為提交推送到 GitHub。
  • 開啟並合併拉取要求。

先決條件

  • 你必須擁有 GitHub 帳號。 有關更多信息，請參閱 在 GitHub 上建立帳戶。

  • 你不需要知道如何編碼、使用命令列或安裝 Git（建立 GitHub 的版本控制軟體）。

第 1 步：建立儲存庫

首先我們需要建立一個儲存庫。 儲存庫就好比包含相關項目的資料夾，例如文件、圖像、視頻，甚至其他資料夾。 儲存庫通常會將屬於相同「專案」或正在處理的交易的項目組合在一起。

通常，儲存庫包括一個 README 文件，其中具有項目的相關資訊。 自述文件是用 Markdown 語言編寫的，這是一種易於讀取和編寫的語言，用於設定純文字格式。

GitHub 可在新儲存庫的同時新增 README 檔案。 GitHub 還提供了其他常用選項，例如許可證文件，但你現在不必選擇其中任何一個。

可在 hello-world 儲存庫中儲存靈感和資源，甚至與他人分享和討論。

  1. 在任何頁面的右上角，選擇+，然後按一下「新建儲存庫」。

  1. 在「儲存庫名稱」方塊中，鍵入 hello-world。
  2. 在「說明」方塊中，鍵入簡短說明。例如，鍵入「此儲存庫用於練習 GitHub 流程」。
  3. 選擇儲存庫是「公共」還是「私有」 。
  4. 選擇「新增 README 檔案」。
  5. 按一下「建立儲存庫」。

第 2 步：建立分支

透過分支，您可以同時擁有不同版本的儲存庫。

預設情況下，儲存庫有一個名為 main 的分支，它被視為最終分支。 可在儲存庫中從 main 建立其他分支。

要在不更改主要程式碼來源的情況下向專案添加新功能時，分支將非常實用。 在合併主分支之前，在不同分支上完成的工作不會顯示在主分支上，我們將在本指南的後面部分介紹。 可使用分支進行試驗和編輯，然後再提交至 main。

從 main 分支建立分支時，建立的是 main 在當時的副本或快照。 如果其他人在你處理分支時對 main 分支進行了更改，你可拉取這些更新。

  • main 分支
  • 名為 feature 的新分支
  • feature 在合併到 main 之前，需要經歷「提交更改」、「提交拉取請求」和「討論建議的更改」階段

建立分支

  1. 按一下 程式碼 標籤以查看您的 hello-world 儲存庫。

  2. 在檔案清單上方，點選main的下拉式選單。

  1. 在文字方塊中鍵入分支名稱 readme-edits。

  2. 點選「建立分支: 從 main 建立 readme-edits」。 ****

現在你有兩個分支：main 和 readme-edits。 現在，它們看起來完全相同。 接下來，您需要在新的 readme-edits 分支中新增變更。

第 3 步：進行和提交更改

在上一個步驟中建立新分支時，GitHub 會將你轉到作為 readme-edits 副本的新 main 分支的程式碼頁。

您可以對儲存庫中的檔案進行變更並儲存變更。 在 GitHub 上，儲存的變更稱為提交。 每個提交都有一個關聯的提交訊息，該訊息是解釋為什麼進行特定更改的說明。 提交訊息會捕獲您更改的歷史記錄，以便其他參與者可以了解您執行了哪些操作及其原因。

  1. 在你建立的 readme-edits 分支下，點選 README.md 檔案。
  2. 若要編輯文件，請點選 。
  3. 在編輯器中，寫一些關於您自己的內容。
  4. 按一下「提交更改」。
  5. 在「提交更改」方塊中，編寫描述更改的提交訊息。
  6. 按一下「提交更改」。

這些變更將僅適用於 readme-edits 分支上的 README 文件，所以此分支現在包含 main 中沒有的內容。

第 4 步：開啟一個拉取請求

現在你已在從 main 建立的分支中進行了更改，接下來可開啟拉取請求。

拉取請求(Pull Request) 是 GitHub 上協作的關鍵。 打開拉取請求後，可以提出更改，要求某人審查和提取您的貢獻並將其合併到其分支中。 拉取請求顯示兩個分支中內容的差異。 變化、增減以不同的顏色顯示。

只要進行提交，便可開啟拉取請求並開始討論，即使在程式碼完成之前亦可。

在這一步驟中，你將在自己的儲存庫中開啟一個拉取請求，然後自行合併。 這是在處理大型專案之前練習 GitHub 流程的好方法。

  1. 按一下儲存庫的 拉取請求 標籤。

  1. 按一下「新建拉取請求」。

  2. 在「範例比較」方塊中，選擇你建立的分支 ****，與 readme-edits（原始分支）進行比較。

  3. 在 Compare（比較）頁面上的差異中查看您的更改，確保它們是您要提交的內容。

  1. 按一下「建立拉取請求」****。

  2. 為拉取請求指定一個標題，並寫下變更的簡要說明。 您可以包含表情符號，並且拖放圖片和GIF。

  3. 按一下「建立拉取要求」****。

審查合併請求

開始與他人協作時，這將是你請求他們進行評審的時間。 這樣就可以在將變更合併到 main 分支之前，讓協作者對拉取請求發表評論或提出更改建議。

第 5 步：合併拉取請求

在最後一步中，你將 readme-edits 分支合併到 main 分支中。 合併拉取請求後，readme-edits 分支上的變更將合併到 main。

有時，拉取請求可能會引入與 main 上的現有程式碼衝突的程式碼變更。 如果有任何衝突，GitHub 將提醒你有關衝突程式碼的信息，並防止合併，直到衝突解決為止。 您可以進行解決衝突的提交，也可以使用拉取請求中的註解與團隊成員討論衝突。

在本演練中，應該沒有任何衝突，因此您已準備好將分支合併到主分支中。

  1. 在拉取請求底部，按一下合併拉取請求以將變更合併到main中。
  2. 按一下「確認合併」。 您將收到一則訊息，指出請求已成功合併且請求已關閉。
  3. 按一下「刪除分支」。 現在你的拉取請求已合併，並且你的更改位於 main 上，接下來你可安全地刪除 readme-edits 分支。 如果要對專案進行更多更改，可以隨時建立新分支並重複此過程。
  4. 按一下返回 **** 儲存庫的「程式碼」hello-world選項卡，以在 main 上查看已發佈的變更。

結束語

透過完成本教程，你已經學會了創建專案和在 GitHub 上發出拉取請求。

在那過程中，我們學會瞭如何：

  • 建立儲存庫。
  • 啟動和管理新分支。
  • 更改文件並將這些更改提交到 GitHub。
  • 開啟並合併拉取要求。

GitHub 獨立金鑰

由於 GitHub 不允許同一個公鑰綁定多個專案，您需要為這兩個專案分別產生不同的 Key（例如 id_rsa_keyA 與 id_rsa_KeyB）。

1. 產生兩組獨立金鑰

  • -t (type): 指定金鑰類型。
  • -f (filename): 指定私鑰存放的檔案路徑和檔名。如果不指定，會使用預設值（通常是 ~/.ssh/id_rsa 或 ~/.ssh/id_ed25519）。
  • -C (comment): 提供一個註解，通常用於辨識使用者（如 email），會顯示在公鑰內容的末尾。
    請將產生的 .pub 公鑰內容分別貼到 GitHub 各自專案的 Settings > Deploy keys 中。

2. 設定 ~/.ssh/config

  • 由於您的 URL 中主機名都是 ://github.com，為了讓 SSH 能根據「路徑」自動選用不同的 Key，最常見的做法是利用 Host 別名。
  • 但如果您堅持要用原生的 URL (不改 alias)，SSH 本身無法根據 URL 的後半段路徑 (<你的github名字/你的專案.git>) 來自動切換金鑰。建議採用以下別名設定方式，並微調您的 Remote URL：
