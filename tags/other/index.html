自解壓縮執行檔設計方案

本方案詳述如何將 demo 專案中所有的目錄結構與檔案，封裝成一個單一的自解壓安裝檔 (demo_setup.exe)。在執行時，此安裝檔會完整還原工作區的目錄結構，隨後自動調用執行目標批次檔 bin\DC.bat。

由於系統環境，本方案 100% 採用 Windows 10/11 內建的原生封裝技術（IExpress 與 PowerShell）。

🛠️ 架構設計與運作原理

本方案由四個核心元件協同運作：

  1. build.bat (建置指令)：在開發環境中執行的自動化打包腳本。
  2. payload.zip (壓縮內容)：包含 demo 所有程式碼、組件與目錄層級的壓縮檔。
  3. launch.bat (自解壓啟動器)：封裝在安裝檔內的引導腳本，負責執行期解壓縮與呼叫主程式。
  4. setup.sed (IExpress 指引設定)：供 Windows 原生封裝工具 iexpress.exe 讀取的設定檔。

📦 元件細節設計

1. 建置指令檔 (build.bat)

位於專案根目錄下，執行時會執行以下操作：

  • 清理舊檔：刪除先前的 payload.zip、launch.bat、setup.sed 與 demo_setup.exe。
  • 打包工作區：調用 PowerShell 將專案目錄的所有檔案與子目錄（排除 .git、建置腳本與設計文件本身）壓縮為 payload.zip，確保目錄層級完整保留。
  • 產生啟動器：自動寫入 launch.bat 至根目錄。
  • 產生配置檔：自動寫入 setup.sed 至根目錄。
  • 執行封裝：調用 %SystemRoot%\System32\iexpress.exe /n /q setup.sed，以無互動背景模式產出 demo_setup.exe。
  • 暫存清理：自動刪除過程中的 payload.zip、launch.bat 與 setup.sed，保持專案乾淨。

2. 啟動器批次檔 (launch.bat)

封裝在 EXE 內的關鍵引導腳本，在解壓至臨時目錄後自動執行：

  • 路徑偵測：支援 %APP_DRIVE% 與 %APP_DIR% 環境變數。若未配置，則預設指向標準路徑 C:\scot\dcdat（此為 DC.bat 預期執行與關聯檔案的正確系統路徑）。
  • 目錄還原：調用 PowerShell 內建的 Expand-Archive 指令，將 payload.zip 解壓縮還原至目標路徑，覆蓋並還原完整的資料夾層級。
  • 主程式調用：切換目錄至 C:\scot\dcdat 並呼叫執行 bin\DC.bat。

📂 建議的程式異動

[新增] build.bat

🧪 驗證與測試計畫

手動測試驗證

  1. 建置測試：
    • 以系統管理員身分執行專案根目錄下的 build.bat。
    • 驗證根目錄下是否成功生成 demo_setup.exe，且無任何編譯報錯。
  2. 安裝與還原測試：
    • 將生成的 demo_setup.exe 放置至乾淨的測試環境中執行。
    • 檢查 C:\demo\ 下是否完整還原了 bin/、config/、devices/ 等所有子資料夾，且檔案大小均正確無誤。
  3. 執行測試：
    • 驗證解壓完畢後，是否已自動開啟終端機並正確執行了 bin\DC.bat 及其內部的所有初始化配置。
